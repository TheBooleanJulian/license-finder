# License templates

Ready-to-use license paperwork for every license this tool can recommend. Pick the folder matching your quiz result — the tool's results page links straight to the right one — and copy its contents into your project's root. Replace any `[bracketed]` placeholder (`[year]`, `[fullname]`, `[Software Name]`, `[Licensor Name]`, `[repository URL]`, `[contact method]`) before publishing.

## Single-license folders

Each of these contains just a `LICENSE` file with the verbatim official text (permissive licenses include `[year]`/`[fullname]` placeholders to fill in; copyleft and source-available licenses have no fillable fields in the license body itself).

## Dual-license folder

| Folder | Model |
|---|---|
| [agpl3-plus-commercial/](agpl3-plus-commercial) | AGPLv3 for the community + a separate paid commercial license for companies that don't want AGPL's obligations |

Dual licensing isn't a single file — the community track just needs the `LICENSE` file from that folder, but the commercial track needs the full paperwork: `NOTICE`, `COMMERCIAL-LICENSE.md`, `COMMERCIAL-LICENSE-AGREEMENT-TEMPLATE.md`, `CLA.md`, `CONTRIBUTING.md`, and `TRADEMARKS.md`. See that folder's own contents for what each file is for.

---

This is a starting point, not legal advice — have a lawyer review the wording before relying on any of this for a real commercial transaction, especially the AGPL+Commercial agreement template.
