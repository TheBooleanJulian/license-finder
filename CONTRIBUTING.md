# Contributing

Thanks for considering a contribution to License Finder.

## Before you start

- For anything beyond a small fix, open an issue first to discuss the change — this avoids wasted work if the approach needs adjusting.
- This is currently a single-file project (`index.html`) with no build step. Keep changes consistent with the existing style: vanilla JS, no framework, no external dependencies beyond the Google Fonts link already in use.

## Submitting changes

1. Fork the repository and create a branch for your change.
2. Test your change by opening `index.html` directly in a browser — there's no build step to run.
3. Open a pull request describing what changed and why.

## Reporting issues

Open an issue on GitHub with enough detail to reproduce the problem — what you expected, what happened instead, and your browser if it's a rendering issue.
